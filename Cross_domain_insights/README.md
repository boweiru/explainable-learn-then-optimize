# Cross-domain systemic insights

This folder contains the data and plotting script used to generate Fig. 6 in the main text. The analysis examines whether representative factors and redundant factors behave differently in the Pareto-optimal solution sets obtained from multiple high-dimensional complex systems.

## What is analysed?

The input data are Pareto fronts (PFs) and Pareto sets (PSs) generated by different multi-objective optimization algorithms across six systems:

| System label in files | System | Nominal dimension `d` | Representative dimension `\|R\|` | Repetitions |
|:--- |:--- |:--- |:--- |:--- |
| `malaria` | Rural malaria transmission | 23 | 5 | 10 |
| `Junction` | Isolated urban junction | 29 | 5 | 10 |
| `Reactor` | Coiled-tube reactor | 50 | 2 | 5 |
| `Aircraft` | Human-powered aircraft | 55 | 5 | 5 |
| `Corridor` | Urban arterial corridor | 60 | 9 | 10 |
| `WFG3` | WFG 3 mathematical function | 100 | 4 | 10 |

For each system, the analysis compares the factor-level dispersion patterns of representative and redundant factors in the PS. The purpose is not to evaluate optimization convergence again, but to ask a structural question: after optimization has produced Pareto-optimal solutions, do the factors identified as representative show a different relationship to the Pareto trade-off structure than the factors identified as redundant?

In this context:

- **PF** refers to the objective-space Pareto front. Each row is one non-dominated objective vector.
- **PS** refers to the decision-space Pareto set. Each row is the parameter vector corresponding to the same row in the paired PF file.
- **Representative factors** are the factors retained by the USED-based learning stage.
- **Redundant factors** are the factors removed from the low-dimensional effective space because their contribution to system-level output variance is negligible.

## Analysis workflow

Run the analysis from the root of this folder:

```bash
python cNDI.py
```

The script reads the PF and PS `.npy` files in `Results/`, computes factor-level cNDI values, performs representative-versus-redundant statistical comparisons, and exports both the final figure and intermediate result tables.

### 1. Pair PF and PS files

For each system, algorithm and repetition, the script loads matched PF and PS files:

```text
Results/PF-<algorithm><dimension>-<system>-repetition-<id>.npy
Results/PS-<algorithm><dimension>-<system>-repetition-<id>.npy
```

The PF and PS files must be aligned row by row: the `i`-th row of the PS file is the decision vector that produces the `i`-th objective vector in the PF file.

### 2. Cluster the PF into local trade-off regions

The Pareto front is first normalized and then clustered in objective space. Each PF cluster represents a local region of similar Pareto trade-offs. This step is important because different regions of a Pareto front correspond to different compromises among objectives; a parameter may legitimately change from one trade-off region to another even when it is structurally important.

### 3. Compute cNDI from the corresponding PS

After PF clustering, the corresponding PS points are grouped according to their PF-cluster labels. The script then computes the Conditioned Normalized Dispersion Index, denoted as **cNDI**, for each factor.

We use **cNDI rather than a global NDI** because multi-objective Pareto solutions are inherently trade-off dependent. A global NDI computed over the entire Pareto set would mix together different objective-compromise regions. This can mistakenly treat meaningful trade-off-driven variation in representative factors as random dispersion. By conditioning on PF clusters, cNDI measures how much a factor varies **within comparable Pareto trade-off regions**, making it better suited to distinguish structured Pareto-set variation from unconstrained redundant-factor drift.

A higher cNDI indicates broader normalized variation of a factor within similar trade-off regions, whereas a lower cNDI indicates stronger localization of that factor within those regions. 

### 4. Compare representative and redundant factors

For each relevant system-algorithm setting, the factor-level cNDI values are divided into two groups:

- cNDI values of representative factors;
- cNDI values of redundant factors.

The script applies a Mann–Whitney U test to compare these two distributions. This nonparametric test is used because factor-level cNDI values are not assumed to be normally distributed and the representative and redundant groups usually have unequal sizes.

The null hypothesis is that representative and redundant factors have the same cNDI distribution. A small p-value indicates that the two groups exhibit statistically distinguishable Pareto-set dispersion patterns. This supports the interpretation that the USED separates factors with different roles in shaping Pareto-optimal system behaviour.

## Input files

### Main script

| File | Meaning |
|---|---|
| `cNDI.py` | Main Python script for loading PF/PS data, clustering PFs, computing cNDI, testing representative versus redundant factors, and generating Fig. 6. |

### Pareto-front and Pareto-set files

All optimization results are stored in `Results/` as NumPy arrays.

| Pattern | Meaning |
|---|---|
| `PF-<algorithm><dimension>-<system>-repetition-<id>.npy` | Objective-space Pareto front for one algorithm, system and independent repetition. Rows are Pareto-optimal objective vectors. |
| `PS-<algorithm><dimension>-<system>-repetition-<id>.npy` | Decision-space Pareto set corresponding to the paired PF file. Rows are Pareto-optimal parameter vectors. |

The algorithm labels are:

| Label | Meaning |
|---|---|
| `NSGA<dimension>` | NSGA-II applied either in the original full-dimensional space or in the reduced representative-factor space. |
| `MOBO<dimension>` | Multi-objective Bayesian optimization applied either in the original full-dimensional space or in the reduced representative-factor space. |
| `XLO` | XLO search guided by the learned low-dimensional effective space. |

## Output files

Running `cNDI.py` generates the following files in the root folder:

| File | Meaning |
|---|---|
| `cNDI.svg` | Final vector figure for Fig. 6. |
| `cNDI_raw_cNDI.csv` | Raw factor-level cNDI records before aggregation. This is the most detailed output and can be used for checking individual systems, algorithms, repetitions, factors and factor groups. |
| `cNDI_summary.csv` | Summary statistics of cNDI by system, algorithm and factor group. |
| `cNDI_rep_vs_red_tests.csv` | Mann–Whitney U test results comparing representative and redundant factors. This file reports the test statistics and p-values used for significance annotation. |
| `cNDI_run_info.csv` | Run-level diagnostic information, including which PF/PS pairs were processed and the available data structure for each run. |
